# Code of conduct

yaka-bridge is intended to be a professional, practical and respectful project.

## Expected behavior

- Be respectful and constructive.
- Assume good intent while staying precise about risks.
- Keep criticism focused on code, documentation, security and product behavior.
- Make room for contributors with different experience levels.
- Protect customer privacy and do not request or post private data.

## Unacceptable behavior

- Harassment, insults or personal attacks.
- Publishing secrets, credentials or private customer information.
- Encouraging insecure production shortcuts.
- Disruptive comments unrelated to the project.
- Misrepresenting unverified security or pricing claims as guaranteed facts.

## Enforcement

Maintainers may edit, hide or remove comments; close issues; block users; or
reject contributions that violate this code of conduct or weaken the project's
security posture.

Report conduct concerns privately to the repository owner.
