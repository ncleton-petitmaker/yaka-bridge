# Bridge Auto Update

Bridge updates are silent in production. The desktop app checks an update feed,
downloads the update automatically, then installs it as soon as no Bridge job is
running.

## Feed Location

Default feed:

```text
<SUPABASE_URL>/storage/v1/object/public/bridge-updates
```

For example:

```text
https://api.customer.example/storage/v1/object/public/bridge-updates
```

Override it at build/runtime when needed:

```bash
export BRIDGE_UPDATE_BASE_URL=https://api.customer.example/storage/v1/object/public/bridge-updates
```

The Control Plane may also return `updateBaseUrl` from `/bridge/sync`; Bridge
persists it and reconfigures the updater after sync.

## Storage Bucket

Create a public bucket for update metadata and signed installers:

```sql
insert into storage.buckets (id, name, public, file_size_limit)
values ('bridge-updates', 'bridge-updates', true, 536870912)
on conflict (id) do update set public = true, file_size_limit = 536870912;
```

This bucket is public because `electron-updater` must fetch metadata without a
Bridge login. Do not store customer data in it.

## Published Files

Upload every file generated by `electron-builder` that is referenced by update
metadata.

Expected minimum:

```text
latest.yml
latest-mac.yml
Bridge-Setup-<version>.exe
Bridge-Setup-<version>.exe.blockmap
Bridge-<version>-mac.zip
Bridge-<version>-mac.zip.blockmap
Bridge-<version>.dmg
Bridge-<version>.dmg.blockmap
```

The manual download bucket may keep stable aliases such as `Bridge.dmg` and
`Bridge-Setup.exe`, but the auto-update bucket should keep immutable versioned
artifacts so older clients can still resolve their expected update metadata.

## Runtime Behavior

- Checks at startup, then periodically.
- Downloads automatically.
- Installs automatically when no Codex/Bridge job is active.
- Uses `autoInstallOnAppQuit` as a backup path if the app is closed manually.
- Is disabled in development unless `BRIDGE_AUTO_UPDATE_IN_DEV=1`.

## Production Requirements

- macOS builds must be signed and notarized.
- Windows builds must be Authenticode signed.
- Update feed must be HTTPS.
- The bucket must contain metadata matching the uploaded artifacts.
- Never publish unsigned smoke-test builds to a customer update feed.
